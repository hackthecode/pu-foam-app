# ПУ Пяна — Производствен Асистент (PWA)
https://hackthecode.github.io/pu-foam-app/

Модерно приложение за изчисления и рецепти при производство на пенополиуретан (дунапрен).
Данните се теглят **на живо** от Google Sheets таблицата при всяко отваряне.

## Какво включва
- **Калкулатор** — материали на пуск и общо + плътност (кг/м³) от размерите и теглото на блока.
- **Калибратор** — двупосочно: обороти (RPM) ↔ количество (грамове), по коефициент от таблицата.
- **Рецепти** — браузър с търсачка по номер / вид / боя и детайлен изглед в кг и грамове.
- **33LV** — амин/катализатор: грамове ↔ пулсове по калибровка гр/пулс.
- **Тестов блок** — мащабира рецепта към точно целево тегло (3/5/10 кг) със запазени пропорции.

## Стартиране на компютър (за тест)
```
node serve.js
```
После отвори: http://localhost:8123

## Инсталиране на Android телефон
1. Качи папката на уеб хостинг с **HTTPS** (напр. GitHub Pages, Netlify, Vercel — безплатни).
   > Важно: за инсталиране и service worker е нужен HTTPS (или localhost).
2. Отвори адреса в **Chrome** на телефона.
3. Меню (⋮) → **„Инсталиране на приложението“ / „Добави към началния екран“**.
4. Приложението се появява с икона и работи на цял екран като нативно.

## Синхронизация на данните
- При всяко отваряне приложението тегли най-новите данни от таблицата (network-first, със заобикаляне на кеша).
- Без интернет работи с последно изтеглените данни (запазени локално).
- Бутонът ↻ горе вдясно обновява данните ръчно.

## Източник на данните
Google Sheets таблица (листове: Recipe, Formula, calibration, 33LV, Notes, Calculator).
Калибровъчните коефициенти за калибратора се четат от листа **Calculator** (колона грамове/RPM).

## Структура на проекта
```
index.html              — обвивка и табове
css/styles.css          — тъмна индустриална тема
js/config.js            — ID на таблицата и URL логика
js/csv.js               — CSV парсер + parseNum (европейски формат)
js/parse.js             — модели от листовете
js/data.js              — live-sync (мрежа + локален кеш)
js/app.js               — логика на 5-те таба
manifest.webmanifest    — PWA манифест
sw.js                   — service worker (кешира само обвивката, НЕ данните)
icons/                  — икони
serve.js                — локален тест сървър
```
